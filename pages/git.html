GIT

Git이란?

소스 관리를 위한 분산 버전 관리 시스템(리눅스 토발즈가 개발)
코드 버전 관리함.
협업 시 필수 요소

깃 명령어

기본적인 git 순서
: ADD - COMMIT - PUSH - PULL


Git Version
: git --version


Clone
: git clone url(ex: https://github.com/ooooo.git)


Commit
commit의 상태
- uncommit : 커밋 안 한 상태(코드만 변경)
- staying : add한 상태(commit하기 위해 올려놓기)
- commit : 변경사항 확정

Add : git add 수정코드패스(ex: git add pages/index.html)
Commint : git commit -m “내용”(ex: git commit -m "index.js 클릭 버그 수정")


Commit reset
: git reset(add한 파일 취소)
: git reset fileName(파일 삭제)
: git rm --cached fileName(파일 삭제, 위와 같은 명령어)
: git reset merge(머지한 코드 취소)


Stash
: git stash(코드 임시 저장, stash 영역에 잠시 저장)
: git stash pop(마지막에 저장한 코드 불러오기)
: git stash list(stash의 목록 확인)


Push
: git push
[pull vs fetch]
- 풀 : git pull(git서버에서 최신 코드 받아와 merge 하기)
- 패치 : git fetch(git서버에서 최신 코드만 받아옴)

pull(merge O, 즉시 반영O - 충돌 발생 가능)
fetch(merge X, 즉시 반영X)


Merge
: git merge branchName1 branchName2(합칠 브랜치의 이름 2개)


Branch 깃 브랜치 확인 : git branch, *(별)표시가 현재 브랜치
깃 브랜치 체크아웃 : git checkout branchName

깃 페이지

레포지토리 이름을 사용자이름.github.io로 만들어 md나 html 파일을 이용하여 하나의 사이트를 만들 수 있다.

*git page를 작성할 때는 .md 파일을 만들 경우 .html이 작동을 안 함. 이중으로 만들면 안 된다.
*깃 페이지를 작성할 때 live Server를 이용하면 빠르게 변경사항을 확인할 수 있다.(liveServer 설치 후 명령어 live-server)
