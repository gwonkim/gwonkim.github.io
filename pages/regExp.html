Regular Expression

정규표현식 이론 정리


특수문자

  • [ ]

    : 괄호 안의 문자 중 일치하는 한 문자

    Bloc[Kk] => BlocK or Block
    0x[0-9A-F][0-9A-F] => 0x2F, 0xEA 등
  • . (점, dot)

    : 앞의 정규식과 일치하는 한 문자

    A.C => A5C, ABC 등
    D..G => DEFG, DegG, D@!G 등
  • [^ ]

    : 괄호 안의 문자를 제외한 모든 문자열 중에서 일치하는 한 문자

    [^Tt]he => Ahe, #he
    [^0-9][a-f] => FG, AC, g3 등
  • ?

    : 앞의 정규식과 일치하는 0 ~ 1개의 문자

    AB?C => ABC, AC
    A[b-g]?C => AC, AeC 등
  • *

    : 앞의 정규식과 일치하는 0개 이상의 문자

    AB*C => AC, ABC, ABBBBBBBC 등
    D[A-Z]*G => DEFG, DEEEEEVG, DG 등
  • +

    : 앞의 정규식과 일치하는 1개 이상의 문자

    AB+C => ABC, ABBBBBBBC 등
    D[A-Z]+G => DAG, DEFG, DEEEEEVG 등
  • ^

    : 각 줄의 선두 부분에서 일치하는 문자열

    ^123 => 선두 부분에 있는 123 문자열과 일치. 줄의 선두가 아니면 일치 X.
  • $

    : 각 줄의 후두 부분에서 일치하는 문자

    ABCD$ => 후두 부분에 있는 ABCD 문자열과 일치. 후두 부분이 아니면 일치 X.
  • |

    :앞뒤 정규식 중 하나 일치(OR)

    Ab|cD => AbD, AcD
    (이 경우 A[bc]D 정규식과 동일.)
    A(and)|(of)D => AandD, AofD
  • ()

    : 괄호 안의 정규식을 묶는 역할

    A(BC)?D => ABCD, AD
    A([a-x][0-3]){3}F => Aa0x1a2F, Ax0a2d2F 등
  • ￦

    : 특수 문자를 보통 문자로 취급할 수 있게 하는 역할

    ￦([1-9][0-9]*￦)=> (1), (1000), (99999) 등
    [￦t] => 탭 문자, 공백 문자와 일치 [￦t]+ => 탭 문자, 공백 문자가 1개 이상 반복된 것과 일치
    (탭탭탭, 공백탭탭탭공백, 탭, 공백, 탭공백탭 등 1개 이상 포함 시 ok)
  • {n}

    : n = number(반드시 1개의 숫자가 입력되어야 함.)
    : 앞의 정규식과 일치하는 문자가 n번 반복

    A[0-9]{2} => A43, A03, A12 등 [0-9]{6}-[1234][0-9]{6} => 주민등록번호 000012-3000555 등
  • {n, m}

    : n = number(2개의 숫자가 입력되어야 함.)
    : 앞의 정규식과 일치하는 문자가 n보다 크거나 같고 m보다 작거나 같게 반복
    : n ≤ x ≤ m

    A[0-9]{2, 6} => A43, A0333, A111333 등
    0[2-9][0-9]?-[0-9]{3,4}-[0-9]{4} => 전화번호 02-2000-4444, 010-0123-4567 등
